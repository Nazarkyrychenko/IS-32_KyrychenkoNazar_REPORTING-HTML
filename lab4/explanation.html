Пояснення

Десктопна версія

Для екранів з шириною від 1024px і більше використовується медіа-запит: @media only screen and (min-width: 1024px). У цій версії:

  • .news-grid: відображається як гнучкий контейнер (flex), що дозволяє компонувати новини в ряд.
  • .footer-content: елементи футера розташовуються горизонтально з відступом у 20px.
  • .search input: поле для пошуку має фіксовану ширину 300px.

Планшетна версія

Для екранів з шириною від 768px до 1023px використовується медіа-запит: @media only screen and (min-width: 768px) and (max-width: 1023px). У цій версії:

  • .news-grid: новини розташовуються в колонку з відступом між ними у 10px.
  • nav ul: елементи навігації розташовуються в ряд з можливістю переносу.
  • .footer-content: елементи футера розташовуються вертикально з відступом у 15px.
  • .hero: додається відступ у 30px, щоб забезпечити кращу читабельність.
  • .search input: поле для пошуку займає всю ширину доступного простору, але не більше ніж 500px.

Мобільна версія

Для екранів з шириною до 767px використовується медіа-запит: @media only screen and (max-width: 767px). У цій версії:

  • nav ul: елементи навігації розташовуються в колонку, вирівнюються зліва для легшого доступу на мобільних пристроях.
  • nav ul li: додаються вертикальні відступи по 5px між елементами списку для зручнішого торкання пальцем.
  • .search: додається верхній відступ у 10px для зручнішого доступу до поля пошуку.
  • .news-grid: новини розташовуються в колонку з відступом у 10px між ними.
  • .footer-content: елементи футера вирівнюються вертикально та вирівнюються зліва для легшої навігації.
  • .hero: додається відступ у 20px, щоб зберегти компактність та читабельність на маленьких екранах.
